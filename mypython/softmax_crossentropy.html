Softmax logits → probabilities

Given raw scores (logits) from the model, softmax converts them into a valid probability distribution — all values positive, all sum to 1.

$$\sigma(\mathbf{z})_i \;=\; \frac{e^{z_i}}{\displaystyle\sum_{j=1}^{K} e^{z_j}}$$
$z_i$ = logit for class $i$,   $K$ = number of classes (vocab size in GPT-2)
Why exponential? $e^x$ is always positive (ensures valid probabilities) and amplifies differences — a logit 2 units higher becomes $e^2 \approx 7.4\times$ more probable. The denominator normalises everything to sum to 1.
$$\sum_{i=1}^{K} \sigma(\mathbf{z})_i = 1, \qquad \sigma(\mathbf{z})_i \in (0, 1)$$
These two properties make the output a valid probability distribution.

Dividing logits by $T$ before softmax controls "sharpness". GPT-2 inference uses this.

$$\sigma\!\left(\frac{\mathbf{z}}{T}\right)_i = \frac{e^{z_i/T}}{\sum_j e^{z_j/T}}$$
Temperature $T$: 1.00

$T = 1$: raw model probabilities

$T > 1$: flatter — more random / creative

Cross-Entropy Loss measures prediction quality

Cross-entropy measures how surprised the model is by the true label. Lower = better.

$$\mathcal{L} \;=\; -\sum_{i=1}^{K} y_i \log p_i$$
General form. $y_i \in \{0,1\}$ is the one-hot label; $p_i$ is the predicted probability.
$$\mathcal{L} \;=\; -\log p_y$$
Simplified: since $y$ is one-hot, all terms vanish except the correct class $y$.
Why $-\log$?   $p_y = 1 \Rightarrow \mathcal{L} = 0$ (perfect).   $p_y \to 0 \Rightarrow \mathcal{L} \to \infty$ (catastrophic). The log scale heavily penalises overconfident wrong predictions.

Softmax + Cross-Entropy fused log-sum-exp form

Plugging softmax into cross-entropy gives a numerically stable form PyTorch uses internally.

$$p_i = \frac{e^{z_i}}{\sum_j e^{z_j}}$$
$$\mathcal{L} = -\log p_y = -\log \frac{e^{z_y}}{\sum_j e^{z_j}}$$
$$\boxed{\mathcal{L} = -z_y + \log \sum_{j=1}^K e^{z_j}}$$
$z_y$ = logit of the correct class · $\log \sum_j e^{z_j}$ = LogSumExp (the normalising term)
Why this form matters:
• No explicit softmax needed — avoids overflow from large $e^{z_i}$
• PyTorch's F.cross_entropy(logits, targets) computes exactly this
• The gradient falls out cleanly (next tab)

The elegant result ∂L/∂z

Differentiating softmax × cross-entropy collapses to a single elegant formula.

$$\frac{\partial \mathcal{L}}{\partial z_i} \;=\; p_i - y_i$$
$p_i$ = predicted probability · $y_i \in \{0,1\}$ = one-hot label
$$\frac{\partial \mathcal{L}}{\partial z_y} = p_y - 1 \;<0$$
Correct class: always push logit up
$$\frac{\partial \mathcal{L}}{\partial z_i} = p_i \;>0 \quad (i \neq y)$$
Wrong classes: push logit down
Intuition: The gradient says "how much did you screw up?" — for the correct token, you were $p_y$ confident but should have been 1.0, so the error is $p_y - 1$. For wrong tokens, you were $p_i$ confident and should have been 0, so the error is $p_i$.

Full chain-rule derivation

We differentiate $\mathcal{L}$ w.r.t. logit $z_i$ via two cases. First, the softmax Jacobian:

$$\frac{\partial p_i}{\partial z_j} = \begin{cases} p_i(1 - p_i) & i = j \\ -p_i\, p_j & i \neq j \end{cases}$$
Derived from the quotient rule on $e^{z_i}/\sum_k e^{z_k}$.
Quotient rule refresher.   For a fraction $f = \dfrac{u}{v}$: $$f' = \frac{u'\,v - u\,v'}{v^2} \quad\text{“(top}'\times\text{bottom} - \text{top}\times\text{bottom}')\,/\,\text{bottom}^2\text{”}$$ Here $u = e^{z_i}$ and $v = S = \sum_k e^{z_k}$. Note cross-entropy is not involved yet — this step only asks how the softmax output $p_i$ moves when we nudge a logit $z_j$.

Where does the Jacobian come from?   Apply the quotient rule to $p_i = e^{z_i}/S$. The only $z_j$-dependent pieces are the numerator $e^{z_i}$ (its derivative is nonzero only when $i=j$) and $S$ (whose derivative is always $e^{z_j}$, since $S$ contains exactly one $e^{z_j}$ term).

$p_i = \dfrac{e^{z_i}}{S}, \quad S = \displaystyle\sum_k e^{z_k}$
$\dfrac{\partial S}{\partial z_j} = e^{z_j}$
setup · derivative of the denominator
$\dfrac{\partial p_i}{\partial z_j}$
$= \dfrac{\dfrac{\partial e^{z_i}}{\partial z_j}\, S \;-\; e^{z_i}\dfrac{\partial S}{\partial z_j}}{S^2}$
quotient rule
$= \dfrac{[i=j]\,e^{z_i} S - e^{z_i} e^{z_j}}{S^2}$
$= [i=j]\,\dfrac{e^{z_i}}{S} - \dfrac{e^{z_i}}{S}\dfrac{e^{z_j}}{S}$
$\partial e^{z_i}/\partial z_j = e^{z_i}$ iff $i=j$, else $0$
$= [i=j]\,p_i - p_i\, p_j$
recognise $e^{z_i}/S = p_i,\; e^{z_j}/S = p_j$
$i = j:\;\; p_i(1 - p_i)$
$i \neq j:\;\; -p_i\, p_j$
split the indicator $[i=j]$
Reading it off: the $-p_i p_j$ term is always present (raising any logit $z_j$ steals probability from class $i$). When $i = j$ there is an extra $+p_i$ from the numerator, giving $p_i - p_i^2 = p_i(1-p_i)$ — raising a class's own logit boosts it, minus the share it loses to renormalisation.
So where does cross-entropy come in? The Jacobian above is only $\partial p_i/\partial z_j$ — softmax differentiated by itself. Cross-entropy is the loss we actually differentiate. Recall it first, then apply the chain rule.

Step 0 — the cross-entropy loss itself (what we're differentiating):

$\mathcal{L}$
$= -\displaystyle\sum_{i=1}^K y_i \log p_i$
cross-entropy, general form
$= -\log p_y$
$y$ is one-hot: $y_y=1$, rest $0$

Step 1 — chain rule splits the loss into a CE part × a softmax part:

Chain rule refresher.   If $\mathcal{L}$ depends on $z_i$ through the intermediate $p_y$ (i.e. $\mathcal{L} = f(p_y)$ and $p_y = g(z_i)$), then differentiate the outer function, then the inner: $$\frac{d}{dx}\,f\big(g(x)\big) = f'\big(g(x)\big)\cdot g'(x), \qquad\text{here}\qquad \frac{\partial \mathcal{L}}{\partial z_i} = \frac{\partial \mathcal{L}}{\partial p_y}\cdot\frac{\partial p_y}{\partial z_i}$$ “outer derivative × inner derivative.” The outer is $\mathcal{L}=-\log p_y$; the inner is the softmax $p_y(z)$.
$$\frac{\partial \mathcal{L}}{\partial z_i} \;=\; \underbrace{\frac{\partial \mathcal{L}}{\partial p_y}}_{\text{outer: CE}}\cdot \underbrace{\frac{\partial p_y}{\partial z_i}}_{\text{inner: softmax}} \;=\; \underbrace{-\frac{1}{p_y}}_{\text{CE derivative}}\cdot \underbrace{\frac{\partial p_y}{\partial z_i}}_{\text{softmax Jacobian}}$$
The outer CE loss contributes the factor $-\tfrac{1}{p_y}$, since $\frac{d}{dx}(-\log x) = -\tfrac1x$. The two cases below plug each Jacobian branch into this — and $p_y$ cancels every time.

Case 1 — correct class ($i = y$):

$\dfrac{\partial \mathcal{L}}{\partial z_y}$
$= \dfrac{\partial}{\partial z_y}\bigl[-\log p_y\bigr]$
definition of $\mathcal{L}$
$= -\dfrac{1}{p_y} \cdot \dfrac{\partial p_y}{\partial z_y}$
chain rule
$= -\dfrac{1}{p_y} \cdot p_y(1 - p_y)$
softmax Jacobian, $i=j$ case
$= p_y - 1$
simplify · always $< 0$

Case 2 — wrong class ($i \neq y$):

$\dfrac{\partial \mathcal{L}}{\partial z_i}$
$= \dfrac{\partial}{\partial z_i}\bigl[-\log p_y\bigr]$
definition of $\mathcal{L}$
$= -\dfrac{1}{p_y} \cdot \dfrac{\partial p_y}{\partial z_i}$
chain rule
$= -\dfrac{1}{p_y} \cdot \bigl(-p_y \cdot p_i\bigr)$
softmax Jacobian, $i \neq j$ case
$= p_i$
simplify · always $> 0$

Unified (both cases at once):

$$\boxed{\frac{\partial \mathcal{L}}{\partial z_i} = p_i - y_i}$$
One formula covers both cases because $y_y = 1$ and $y_i = 0$ for $i \neq y$.
The $p_y$ cancels perfectly in both cases — that's why the combined softmax + cross-entropy gradient is so clean. Differentiating them separately is much messier; together they simplify beautifully.

Quotient Rule differentiating a fraction

When you have a function written as one thing divided by another — like softmax's $e^{z_i}/S$ — this is the rule for its derivative.

$$f(x) = \frac{u(x)}{v(x)} \quad\Longrightarrow\quad f'(x) = \frac{u'\,v - u\,v'}{v^2}$$
$u$ = top (numerator),   $v$ = bottom (denominator)
How to remember it:   “low d-high minus high d-low, over low-squared.”

(top′ × bottom) − (top × bottom′), all divided by bottom².

Order matters — the $u'v$ term comes first (the minus sign would flip the answer if you swapped them).

Worked example

Take a concrete fraction and apply the rule step by step.

$f(x) = \dfrac{x^2}{x+1}$
$u = x^2,\quad v = x+1$
identify top & bottom
$u' = 2x$
$v' = 1$
differentiate each part
$f' = \dfrac{u'v - uv'}{v^2}$
$= \dfrac{2x(x+1) - x^2(1)}{(x+1)^2}$
plug into the rule
$= \dfrac{x^2 + 2x}{(x+1)^2}$
simplify the top

Why softmax needs it connection

Softmax is literally a fraction, so its derivative is a quotient-rule problem.

$$p_i = \frac{\overbrace{e^{z_i}}^{u}}{\underbrace{\textstyle\sum_k e^{z_k}}_{v\,=\,S}}$$
Here $u = e^{z_i}$ and $v = S = \sum_k e^{z_k}$. Applying exactly the rule above (with $u' = \partial e^{z_i}/\partial z_j$ and $v' = \partial S/\partial z_j = e^{z_j}$) gives the softmax Jacobian $\partial p_i/\partial z_j$ shown in the Gradient tab. Nothing new — same rule, just $u,v$ replaced by exponentials.

Chain Rule differentiating nested functions

When a function is wrapped inside another — like $\mathcal{L} = -\log(\,p_y(z)\,)$, a loss of a probability of a logit — this is how you differentiate through the layers.

$$\frac{d}{dx}\,f\big(g(x)\big) = f'\big(g(x)\big)\cdot g'(x)$$
$f$ = outer function,   $g$ = inner function
How to remember it:   “derivative of the outside × derivative of the inside.”

Peel it like an onion: differentiate the outermost layer first (leaving the inside untouched), then multiply by the derivative of the inside. Keep going for deeper nesting.

Worked example

Differentiate $y = (2x+1)^3$ — a cube (outer) wrapped around a line (inner).

$y = (2x+1)^3$
outer $f(u)=u^3$,   inner $u=2x+1$
name the layers
$f'(u) = 3u^2$
$g'(x) = 2$
differentiate each layer
$\dfrac{dy}{dx} = f'(u)\cdot g'(x)$
$= 3(2x+1)^2 \cdot 2$
outer′ × inner′
$= 6(2x+1)^2$
simplify

Why cross-entropy needs it connection

The loss is a function of a function of a function — three nested layers.

$$\mathcal{L} = -\log\big(\,\underbrace{p_y}_{\text{inner}}\,\big), \qquad p_y = \underbrace{\text{softmax}(\mathbf{z})_y}_{\text{depends on logits}}$$
$\dfrac{\partial \mathcal{L}}{\partial z_i} = \dfrac{\partial \mathcal{L}}{\partial p_y}\cdot\dfrac{\partial p_y}{\partial z_i}$
$= \underbrace{-\dfrac{1}{p_y}}_{\text{outer: }-\log}\cdot\underbrace{\dfrac{\partial p_y}{\partial z_i}}_{\text{inner: softmax}}$
outer′ × inner′
The outer layer $-\log(\cdot)$ contributes $-\tfrac{1}{p_y}$; the inner layer (softmax) contributes its Jacobian (computed with the Quotient Rule tab). Multiply them and $p_y$ cancels, leaving the famous $\partial\mathcal{L}/\partial z_i = p_i - y_i$. This is the Step 1 you see in the Gradient tab.

Learning Rate how big a step to take

Once you have the gradient (e.g. $p_i - y_i$), it only tells you the direction to move. The learning rate $\eta$ decides how far you step in that direction.

$$\theta \;\leftarrow\; \theta \;-\; \eta \,\frac{\partial \mathcal{L}}{\partial \theta}$$
$\theta$ = a parameter (weight) · $\eta$ = learning rate (a small positive number) · $\partial\mathcal{L}/\partial\theta$ = gradient
Why subtract? The gradient points uphill (direction of increasing loss). We want to decrease loss, so we step the opposite way — hence the minus sign. This is gradient descent.

Why scale by $\eta$? The raw gradient can be huge or tiny. $\eta$ is the knob that turns it into a sensible step size.

Too small · just right · too large

$\eta$What happensPicture
too smallLoss creeps down very slowly — training takes forever, may stall.🐌 tiny steps
just rightLoss drops smoothly and quickly to the minimum.✅ smooth descent
too largeSteps overshoot the minimum — loss bounces, oscillates, or blows up to ∞.💥 diverges
A learning rate that is too high is the #1 cause of training that "explodes" (loss → NaN). When in doubt, lower it.

Connection to softmax + cross-entropy the full loop

Putting every tab together, one training step is:

1. Forward: logits $\mathbf{z}$ → softmax → $p_i$ → loss $\mathcal{L} = -\log p_y$
2. Backward: gradient $\dfrac{\partial \mathcal{L}}{\partial z_i} = p_i - y_i$  (Quotient + Chain rule)
3. Update: every weight $\theta \leftarrow \theta - \eta\,\dfrac{\partial \mathcal{L}}{\partial \theta}$

The gradient $p_i - y_i$ flows back through all layers; $\eta$ controls how big each weight nudge is.
GPT-2 / nanoGPT typically use $\eta \approx 3\times10^{-4}$ (with AdamW), plus a warmup (ramp $\eta$ up from 0) and cosine decay (ramp it back down) over training. Modern optimisers like Adam also adapt the effective step size per-parameter, but $\eta$ is still the master scale.
